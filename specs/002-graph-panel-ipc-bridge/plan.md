# Implementation Plan: OntoGraph-lite Graph Panel & IPC Bridge

**Branch**: `002-graph-panel-ipc-bridge` | **Date**: 2026-06-03 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/002-graph-panel-ipc-bridge/spec.md`

## Summary

Wire the bidirectional IPC bridge between the OntoGraph Editor authoring panel and OntoGraph-lite's graph panel. OntoGraph-lite is a companion VS Code extension — no webview embedding needed. The bridge uses VS Code commands as the cross-extension IPC channel. Two small additions to OntoGraph-lite plus three changes to our extension host complete the integration.

## Technical Context

**Language/Version**: TypeScript 5.x (both extensions)

**Primary Dependencies**: VS Code API 1.90+, OntoGraph-lite extension (`ysgao.ontograph-lite`)

**Storage**: N/A — stateless command-based IPC

**Testing**: Manual F5 debug session with both extensions active; `tsc --noEmit` as type gate

**Target Platform**: VS Code Desktop 1.90+ (macOS/Windows/Linux)

**Project Type**: VS Code Extension (companion extension pattern)

**Performance Goals**: IPC round-trip < 500ms (spec SC-003, SC-004) — command dispatch is synchronous in VS Code, overhead is negligible

**Constraints**: OntoGraph-lite graph shows data only from loaded OWL ontology — CONCEPT_FOCUS navigates to concept if present in model, otherwise falls back to root view (acceptable v1 behavior)

**Scale/Scope**: 2 files patched in OntoGraph Editor, 2 files patched in OntoGraph-lite, 1 new contract file

## IPC Bridge Architecture

```
AuthoringPanel (webview)
   |  postMessage({ command: 'CONCEPT_FOCUS', payload: { id, label } })
   v
authoringPanel.ts:handleMessage()
   |  vscode.commands.executeCommand('ontographEditor.ipcRoute', message)
   v
extension.ts:ipcRoute handler
   |  translateSctid → 'http://snomed.info/id/404684003'
   |  vscode.commands.executeCommand('ontograph.focusEntity', iri)
   v
OntoGraph-lite:focusEntity command
   |  openGraphView(context, model, iri)
   v
OntoGraph-lite graph panel (existing webview)

                             ──────────── reverse ────────────

OntoGraph-lite graph panel
   |  nodeClicked { iri: 'http://snomed.info/id/404684003' }
   v
openVisualization.ts:nodeClicked handler (PATCHED)
   |  extractSctid(iri) → '404684003'
   |  vscode.commands.executeCommand('ontographEditor.ipcRoute', { command: 'GRAPH_NODE_SELECT', payload: { id } })
   v
extension.ts:ipcRoute handler
   |  AuthoringPanel.postMessage(message)
   v
AuthoringPanel (webview) receives GRAPH_NODE_SELECT
```

## Implementation Phases

### Phase A: IPC Contract File

- A1. Copy `specs/002-.../contracts/ipc-messages.ts` to `extension/src/ipcMessages.ts`
- A2. `tsc --noEmit` in `extension/` must pass with the new file

### Phase B: AuthoringPanel Patch

- B1. Add `import { IpcMessage } from './ipcMessages'` to `authoringPanel.ts`
- B2. Add `static postMessage(msg: IpcMessage): void`:
  - Guard: if `!AuthoringPanel.instance` return
  - Body: `AuthoringPanel.instance.panel.webview.postMessage(msg)`

### Phase C: Extension.ts Wiring (OntoGraph Editor)

- C1. Add imports: `IpcMessage`, `isConceptFocus`, `isGraphNodeSelect` from `./ipcMessages`
- C2. Register `ontographEditor.openGraph`:
  ```ts
  vscode.commands.executeCommand('ontograph.openGraph')
  ```
- C3. Register `ontographEditor.ipcRoute`:
  - `isConceptFocus(msg)` → translate `msg.payload.id` to `http://snomed.info/id/${id}` → call `ontograph.focusEntity(iri)` with silent error swallow
  - `isGraphNodeSelect(msg)` → `AuthoringPanel.postMessage(msg)`

### Phase D: Extension package.json

- D1. Add `"extensionDependencies": ["ysgao.ontograph-lite"]` to `extension/package.json`

### Phase E: OntoGraph-lite Patches (apps/OntoGraph-lite submodule)

- E1. `src/extension.ts`: register `ontograph.focusEntity` command that calls `openGraphView(context, model, iri)`
  - Model reference must be accessible at command registration time (check existing model variable scope)
- E2. `package.json`: add `ontograph.focusEntity` to `contributes.commands`
- E3. `src/commands/openVisualization.ts`: patch `nodeClicked` handler:
  - Add `extractSctid(iri: string): string | undefined` helper
  - Replace `// Nothing for now` with command dispatch to `ontographEditor.ipcRoute`
  - Guard dispatch with `.then(undefined, () => {})` so it's silent when OntoGraph Editor is absent

### Phase F: Verification

- F1. `tsc --noEmit` in both `extension/` and `apps/OntoGraph-lite/` — no errors
- F2. F5 debug session with OntoGraph-lite also loaded — both commands appear in palette
- F3. `ontographEditor.openGraph` command → OntoGraph-lite graph panel opens
- F4. Select concept in authoring panel → OntoGraph-lite graph navigates to concept
- F5. Click node in OntoGraph-lite graph → authoring panel receives GRAPH_NODE_SELECT (verify via console log)
