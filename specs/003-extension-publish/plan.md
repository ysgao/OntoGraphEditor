# Implementation Plan: Extension Packaging and Marketplace Publication

**Branch**: `003-extension-publish` | **Date**: 2026-06-03 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/003-extension-publish/spec.md`

## Summary

Package the OntoGraph Editor VS Code extension as a distributable `.vsix` file and publish it to the VS Code Marketplace. Key additions: JRE 21+ detection at activation (with actionable error notification), `.vscodeignore` to keep VSIX under 50 MB, enhanced extension manifest metadata, an "Install OntoGraph-lite" action for the missing-dependency case, and a GitHub Actions CI/CD pipeline that publishes on version tag push.

`extensionDependencies: ["ysgao.ontograph-lite"]` is already declared in `extension/package.json` — no change needed for FR-002.

## Implementation Phases

### Phase A — JRE Detection Service

**Goal**: New `jreDetector.ts` service + activation hook. Satisfies FR-004, SC-004.

1. Create `extension/src/jreDetector.ts` implementing `IJreDetector`:
   - `detect()`: call `child_process.spawnSync('java', ['-version'], { encoding: 'utf8', timeout: 3000 })`
   - Parse stderr with regex `/version "(\d+)/` to extract major version
   - Handle spawn error (ENOENT = not found), timeout, and unexpected output
   - Return `JreDetectionResult` (never throw)

2. Modify `extension/src/extension.ts` `activate()`:
   - Import and call `new JreDetector().detect()` early in activation
   - If `!result.compatible`:
     - `found: false` → show error: `"OntoGraph Editor requires Java 21 or later. [Install Java]"` with URI action to `JRE_DOWNLOAD_URL`
     - `found: true, major < 21` → show error: `"OntoGraph Editor requires Java 21+. Found Java {major}. [Install Java]"` with URI action
   - Use `vscode.window.showErrorMessage` with action button

**Test**: Activate extension without JRE on PATH → error notification appears within 3s. Activate with JRE 21+ → no notification.

---

### Phase B — "Install Now" Companion Action

**Goal**: Enhance existing `openGraph` warning to offer a one-click install action. Satisfies FR-008.

1. Modify the `ontographEditor.openGraph` error handler in `extension.ts`:
   - Change from plain `showWarningMessage(string)` to `showWarningMessage(string, 'Install OntoGraph-lite')`
   - On button click: `vscode.commands.executeCommand('workbench.extensions.installExtension', 'ysgao.ontograph-lite')`

**Test**: With OntoGraph-lite uninstalled, invoke "Open Visualization Display" → notification with "Install OntoGraph-lite" button appears and triggers marketplace install.

---

### Phase C — Manifest Metadata and Packaging Config

**Goal**: Production-ready `package.json` + `.vscodeignore` + `@vscode/vsce` devDep. Satisfies FR-001, FR-003, FR-005, FR-006, SC-001, SC-005.

1. Update `extension/package.json`:
   - `"version": "1.0.0"` (first public release)
   - `"description": "Unified SNOMED CT ontology authoring and visualization environment for VS Code — pairs the AuthoringUI editing workbench with OntoGraph-lite graph visualization."`
   - `"categories": ["Education", "Other"]`
   - `"icon": "assets/icon.png"`
   - `"repository": { "type": "git", "url": "<actual GitHub remote URL>" }`
   - Add `"@vscode/vsce": "^3.0.0"` to `devDependencies`
   - Add `"package:vsix": "vsce package --out dist/"` to `scripts`

2. Update root `package.json`:
   - Add `"package:vsix": "npm run package:vsix --prefix extension"` to `scripts`

3. Create `extension/assets/` directory with a 128×128 PNG placeholder icon.

4. Create `extension/.vscodeignore` (see research.md Decision 5 for full exclusion list).

**Test**: Run `npm run package:vsix` from repo root → `.vsix` produced in `extension/dist/`. File size < 50 MB. Clean install via "Install from VSIX" → both commands available and functional.

---

### Phase D — CI/CD Release Pipeline

**Goal**: GitHub Actions workflow for automated build + publish on version tag. Satisfies FR-007, SC-003.

1. Create `.github/workflows/release.yml` with stages:
   - `actions/checkout@v4` with `submodules: recursive`
   - `actions/setup-node@v4` with Node 18 and npm cache
   - `npm ci` (root) + `npm ci --prefix extension`
   - `npm run build:client` (Angular)
   - `npm run build:extension` (esbuild)
   - `npm test` (extension tests)
   - `HaaLeo/publish-vscode-extension@v1` with `VSCE_PAT` secret

2. Document `VSCE_PAT` secret requirement in `quickstart.md` (already done).

**Test**: Push `v1.0.0` tag → Actions pipeline completes all stages → extension version `1.0.0` visible on marketplace.

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Angular `dist/` not compiled when VSIX packaged | VSIX missing Angular assets | `package:vsix` must run after `build:client`; CI enforces ordering |
| Icon file absent at package time | `vsce package` errors | Create placeholder `icon.png` in Phase C before any packaging |
| `VSCE_PAT` missing in CI | Publish step fails | Document in `quickstart.md`; CI fails with actionable error |
| `java -version` output format varies by JRE vendor | Major version parsed as `undefined` | Regex handles `"1.X"` and `"XX.Y"` formats; fallback to warning not error |
| `ysgao.ontograph-lite` not yet published | `extensionDependencies` auto-install fails silently | Out of scope per spec assumption; users install manually until published |
