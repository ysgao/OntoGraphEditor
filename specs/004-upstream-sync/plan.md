# Implementation Plan: Upstream Submodule Sync Workflow

**Branch**: `004-upstream-sync` | **Date**: 2026-06-03 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/004-upstream-sync/spec.md`

## Summary

Establish a safe, repeatable upstream sync workflow for maintainers of OntoGraph Editor. The two submodules — `apps/authoring-ui-vscode` (fork of IHTSDO/authoring-ui) and `apps/OntoGraph-lite` (unmodified upstream) — must be kept current with their respective origins. The primary challenge is that `apps/authoring-ui-vscode` carries VS Code-specific customizations that must survive every upstream merge. Deliverables: upstream remote configuration, a documented sync runbook (`quickstart.md`), and a build-verification step.

## Technical Context

**Language/Version**: Shell (bash), git 2.x

**Primary Dependencies**: git, Node.js 18+ (for build verification)

**Storage**: N/A — workflow is purely git-based

**Testing**: Manual smoke test — `npm run build-all` exit 0 after sync

**Target Platform**: Developer workstation (macOS / Linux)

**Project Type**: Developer tooling / maintenance workflow

**Performance Goals**: Full sync (both submodules + build) completes in under 30 minutes

**Constraints**: VS Code customization files (`vscode.service.ts`, `HashLocationStrategy` config) MUST survive every merge; no automated CI scheduling in this iteration

**Scale/Scope**: Two submodules, one runbook, ~7 functional requirements
